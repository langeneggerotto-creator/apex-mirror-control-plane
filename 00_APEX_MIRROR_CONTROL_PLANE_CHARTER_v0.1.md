## Purpose Kernel

**APEX Mirror exists to show what is true, what is being built, what is blocked, what benefit is expected, and what Otto may approve, switch, hold, stop or redirect at any time.**

## Core Responsibilities

1. Display module purpose, current maturity, risk, evidence and next valid action.
2. Surface the Final Direction Stack for every active module.
3. Support a visible `STAY / SWITCH / HOLD / STOP / INNOVATE` decision mechanism.
4. Use Benefit Governor scoring to compare payoff, risk, evidence, burden and reversibility.
5. Connect to the Evidence Ledger before displaying promotion claims.
6. Keep human approval and truth status visible at all times.

## Foundational Decision Modes

| Mode | Meaning |
|---|---|
| `STAY` | Continue current approved build path. |
| `SWITCH` | Redirect to a higher-value evidenced direction. |
| `HOLD` | Pause while evidence, safety or ownership is incomplete. |
| `STOP` | End an unproductive, unsafe or invalid path. |
| `INNOVATE` | Explore a sandboxed alternative without promoting it as active. |

## Initial Components

## Public Repository Boundary

This public repository may contain architecture, generic UI prototypes, public-safe schemas and test fixtures. It must not contain confidential enterprise data, private approval records, credentials or sensitive evidence.

## Final Direction Stack

1. Build a public-safe module registry connected to evidence receipts.
2. Define Benefit Governor scoring and switch/stay decision logic.
3. Prototype the APEX Mirror control dashboard and approval controls.
4. Sandbox a Repository Constellation interface showing maturity and dependency paths.
